etcd 백업 / 복원 — 시험 단골 ⭐ 외울 때까지 반복
스냅샷 백업
시험 단골▾
ETCDCTL_API=3 etcdctl snapshot save /backup/etcd.db \
  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key
백업 경로는 문제에서 지정해줌. 인증서는 항상 /etc/kubernetes/pki/etcd/ 안에 있음.
백업 검증
확인▾
ETCDCTL_API=3 etcdctl snapshot status /backup/etcd.db --write-out=table
스냅샷 복원
시험 단골▾
ETCDCTL_API=3 etcdctl snapshot restore /backup/etcd.db \
  --data-dir=/var/lib/etcd-restore
복원 후 /etc/kubernetes/manifests/etcd.yaml 의 --data-dir 경로도 수정 필요!

RBAC 4가지 조합 — 전부 외우기
① User + Role (네임스페이스 범위)
시험 단골▾
kubectl create role <role> --verb=get,list,watch --resource=pods -n <ns>
kubectl create rolebinding <rb> --role=<role> --user=<user> -n <ns>
② User + ClusterRole (클러스터 범위)
시험 단골▾
kubectl create clusterrole <cr> --verb=get,list --resource=pods
kubectl create clusterrolebinding <crb> --clusterrole=<cr> --user=<user>
③ ServiceAccount + Role
SA▾
kubectl create serviceaccount <sa> -n <ns>
kubectl create rolebinding <rb> --role=<role> --serviceaccount=<ns>:<sa> -n <ns>
④ ServiceAccount + ClusterRole
SA▾
kubectl create clusterrolebinding <crb> --clusterrole=<cr> --serviceaccount=<ns>:<sa>

PersistentVolume / PersistentVolumeClaim
PV YAML 기본 구조
PV▾
apiVersion: v1
kind: PersistentVolume
metadata:
  name: my-pv
spec:
  capacity:
    storage: 1Gi
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  hostPath:
    path: /data/pv
PVC YAML 기본 구조
PVC▾
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
  storageClassName: ""
accessModes 3가지
암기▾
ReadWriteOnce  (RWO) — 단일 노드 읽기/쓰기
ReadOnlyMany   (ROX) — 다수 노드 읽기만
ReadWriteMany  (RWX) — 다수 노드 읽기/쓰기
트러블슈팅 순서 — Pod → Node → Control Plane

2단계: Node 상태 확인
2단계▾

journalctl -u kubelet -n 50 --no-pager

3단계▾
kubectl get pods -n kube-system
kubectl logs kube-apiserver-<node> -n kube-system
ls /etc/kubernetes/manifests/
cat /etc/kubernetes/manifests/kube-apiserver.yaml
